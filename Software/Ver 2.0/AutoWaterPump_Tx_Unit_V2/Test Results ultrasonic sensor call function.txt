# Ultrasonic Sensor Call Test Results
Date: December 10, 2025
Author: Yadukrishnan K M
Platform: Arduino Uno, IDE 2.3.6
Sensor: DYP-A02YYTW-V2.0, software UART
Test Condition: 5V, 16Mhz clk, Room temp.
***************************************************************************************

## Summary
Tested `ultrasonicSensorCall` with normal & error conditions. All passed without halts.
Robust retries; debug inflates times (up to 6x). Untested errors (2,3) expected to work similarly.

**Worst Case:** Error 5 + full debug = 6457 ms.

## Execution Times (ms)
| Condition       | Error | No Debug | Full Debug | Retries | Status  |
|-----------------|-------|----------|------------|---------|---------|
| Normal          | None  | 302      | 1127       | 0       | Passed  |
| Sim. Comm Fail  | 1     | 1514     | 3102       | 10      | Passed  |
| Sim. Out-of-Rng | 4     | 755      | 2153       | 5       | Passed  |
| Sim. Unstable   | 5     | 1814     | 6457       | 6       | Passed  |

Measured via `millis()` around function calls.

Errors simulation technique:
- Err 1: Disconnect data or power supply to the sensor.
- Err 4: Place object too close (<3 cm blind zone).
- Err 5: Move hand very fast above the sensor.

## Troubleshooting
- **Err 1 (Comm Fail):** Check sensor power (3.3-5V stable) & data lines (TX/RX wiring, no EMI).

## Improvements
- Add watchdog timer (e.g., `wdt_enable(WDTO_8S)`) for >8s hangs.
- Tune thresholds (e.g., UNSTABLE_READING_THRESHOLD=3) for faster recovery.
- Test remaining errors (2: Header, 3: Checksum) next.

Program fault-tolerant for industrial use. Contact for code/watchdog integration.
